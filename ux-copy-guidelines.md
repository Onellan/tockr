# UX Copy Guidelines

- Prefer `Client` in UI copy; keep `customer` in routes and data model.
- Prefer `Work Type` when explaining activities to users.
- Describe tasks as deliverables or work packages.
- Use billing language that reduces uncertainty:
  - `Billable`
  - `Internal`
  - `Pending`
  - `Unbilled`
  - `Billing attention`
- Keep admin screens explicit and calm; explain consequences before recalculation or financial changes.
- Avoid generic SaaS language like `directory`, `productivity`, or `engagement` when a consulting-engineering term is clearer.
