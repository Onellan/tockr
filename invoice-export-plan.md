# Invoice and Export Plan

## Current state
- Invoice creation: scans unexported billable timesheets for a customer+date range, writes a 3-line HTML stub.
- Invoice download: serves the HTML file directly.
- No line items shown in the download.
- No customer info in the download.

## Richer HTML invoice template

### Data included
- Workspace name (from settings or workspace record)
- Invoice number, date, status
- Customer: name, company, email
- Line items table: description, duration (hours), rate (per hour), line total
- Subtotal, tax (if any), grand total
- Currency label
- Footer: "Generated by Tockr"

### Implementation
`writeInvoiceFile(inv *domain.Invoice)` becomes `writeInvoiceFile(inv *domain.Invoice, customer domain.Customer, items []domain.InvoiceItem, workspaceName string)`

The server handler calls `store.InvoiceDetails(ctx, id)` which returns invoice + items. Customer is fetched from the store. Workspace name comes from the session/access context.

### Backward compatibility
- Existing invoice records can be re-downloaded. The HTML file is regenerated on next download if the file is missing or can be regenerated by the admin (future).
- Currently the file is written at invoice creation time; existing invoices keep their stub until regenerated.
- NEW invoices created after this change get the full template.

## CSV exports

### Reports CSV
- Endpoint: `GET /reports/export` (requires `PermViewReports`)
- Same query params as reports page
- Content-Type: `text/csv`
- Columns: `group_value, entry_count, duration_seconds, revenue_cents`

### Timesheets CSV  
- Endpoint: `GET /timesheets/export` (scoped to user/workspace like timesheets page)
- Columns: `id, started_at, ended_at, duration_seconds, break_seconds, user_id, customer_id, project_id, activity_id, task_id, billable, exported, rate_cents, internal_rate_cents, description`
- Respects same filter/scope as timesheets page

### No XLSX for now
- Would require an external dependency.
- Deferred. CSV is sufficient for import into Excel/Sheets.

## Future export improvements (deferred)
- XLSX via a lightweight library if demand grows.
- PDF via headless browser or wkhtmltopdf — deferred per architecture constraints.
- Scheduled export delivery — deferred.
