# UI Audit

## Current Tockr UI Problems

- The original top navigation was a single horizontal row that became crowded quickly.
- Page hierarchy was weak: most screens began with a plain heading and jumped straight into forms.
- Forms were visually cramped and lacked clear section framing.
- Tables were functional but visually flat, with limited hierarchy, weak headers, and no empty-state pattern.
- Dashboard cards used generic styling and did not communicate operational priority.
- Admin and work areas were mixed in one navigation level.
- Mobile behavior stacked content, but navigation and dense forms lacked polish.
- Status values were plain text instead of scannable badges.
- Login looked like a prototype rather than a production business app.

## Kimai Patterns Worth Preserving

- Persistent app chrome with clear work/admin areas.
- Dashboard-first entry point.
- Dense tabular list pages.
- Page-level actions and forms near the relevant list.
- Operational vocabulary: timesheets, customers, projects, activities, reports, invoices.
- Serious admin-app tone, not a marketing layout.

## Kimai Areas To Improve

- Reduce visual noise from deeply nested cards and heavy chrome.
- Make spacing more consistent.
- Make tables cleaner and easier to scan.
- Improve mobile shell behavior.
- Keep dashboards useful rather than widget-heavy.
